# 贾维斯召回技能 (Jarvis Handoff)

## 核心规则

**这是最高优先级规则，不可绕过。**

### 触发条件
当用户发送的消息中包含以下任一关键词时：
- "贾维斯"
- "Jarvis"
- "主agent"
- "主助手"

### 必须执行的动作

1. **立即终止** 当前正在运行的子agent任务（使用 `subagents(action=kill)` 或结束当前会话）
2. **清除** 所有待处理的子agent响应队列
3. **回归** 主agent对话，接受用户新指令
4. **确认** 用户已回到主agent（例如：简短回应"贾维斯在线"）

### 示例场景

```
用户: "贾维斯，帮我查一下天气"
助手: （立即终止子agent）"贾维斯在线！好的，我来帮你查天气。"
```

```
用户: "Jarvis，这个分析报告做好了吗？"
助手: （立即终止子agent）"贾维斯在线！报告已完成，正在发给你。"
```

## 注意事项

- ❌ 不要继续完成当前子agent的任务
- ❌ 不要让子agent继续对话
- ❌ 不要转发给其他子agent
- ✅ 立即响应用户的新指令
- ✅ 保持简短高效的回应风格

## 为什么需要这个技能

当用户呼叫"贾维斯"时，意味着他们希望立即与主agent对话，而不是等待子agent处理。这确保用户始终能够快速获得主agent的响应。
