# map-route-reader

## 产品定位
把“地图路线查询”做得更像结果型产品的 skill：不是只说个大概时间，而是尽量把真实路线读出来，整理成用户可以立刻参考的出行方案。

## 一句话卖点
直接读取高德 / 百度地图上的真实路线结果，优先返回用户能照着走的公交、地铁、打车或步行方案。

## 为什么会更想下载
很多路线类能力的问题不是不会说，而是：
- 说得很像懂，但答案不够落地
- 只给大概时长，不给真实换乘结构
- 看起来高级，实际装起来麻烦

这个 skill 更值得装的点在于：
- 优先面向真实地图结果
- 优先给可执行路线，不只给大概时间
- 支持路线对比，不只拍脑袋判断
- 更适合“现在就要知道怎么走”的场景

## 适用场景
- 怎么从 A 到 B
- 走哪条地铁更好
- 打车是不是更快
- 哪条路线换乘更少
- 哪个方案更省走路

## 和普通路线回答的区别
很多回答停在：
- 差不多多久
- 可能怎么走
- 应该哪条线更方便

这个 skill 的目标是：
- 读真实路线
- 给出换乘、站点、步行段
- 用户问“哪个更好”时，真的把方案拉出来比较
